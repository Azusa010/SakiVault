前端实习作品集项目文档 · 2025

项目概览

AniVault 是一个基于 AniList GraphQL API
的番剧发现与收藏应用。和管理后台不同，这个项目偏向 C
端用户体验：展示真实数据、处理异步状态、动态路由跳转、本地数据持久化。两个项目放在一起，能覆盖前端实习最常见的两种业务场景。

为什么用 AniList API

- 完全免费，无需注册或申请 Key，直接请求即可

- 支持 CORS，前端可以直接调用，不需要后端代理

- GraphQL 接口：可以精确指定需要哪些字段，减少数据冗余------GraphQL
  经验写进简历也是加分点

- 数据丰富：番剧名称、封面图、评分、简介、剧集数、播出季度等信息都有

- 接口地址：https://graphql.anilist.co，POST 请求，body 是 GraphQL query
  字符串

功能模块详细说明

模块一：首页（/）

首页是流量入口，要有视觉冲击力，同时展示两个数据区块。

Hero 区域

- 大标题 + 副标题（\"发现你的下一部神作\"之类）

- 搜索框：输入关键词按回车或点击搜索按钮跳转到 /search?q=关键词

当季新番列表

- 调用 AniList API 获取当前季度正在播出的番剧（season + seasonYear
  参数）

- 卡片网格展示：封面图 + 番剧名（中文优先，无中文则日文）+ 评分 + 剧集数

- 卡片 hover 效果：上移阴影 + 封面图微放大（纯 CSS transition）

- 骨架屏：数据加载期间显示灰色占位块（用 CSS 动画模拟 shimmer 效果）

- 点击卡片跳转到 /anime/:id 详情页

热门番剧列表

- 调用 API 获取全站最高人气番剧 Top 10

- 样式与当季新番区块相同，但标题不同

模块二：搜索页（/search）

从 URL query 参数 q
读取关键词，自动触发搜索，支持在搜索页继续修改关键词。

- 搜索框回显当前关键词（从 route.query.q 获取）

- 调用 AniList API 的 search 接口，传入关键词

- 结果列表：同首页卡片样式

- 无结果时显示空状态提示：\"没有找到相关番剧，换个关键词试试？\"

- 加载中显示骨架屏

- 关键词变化时（监听 route.query.q）自动重新请求

模块三：详情页（/anime/:id）

动态路由页面，这是体现路由使用能力的核心页面。

顶部大图区域

- Banner 图：用番剧的 bannerImage 作为背景，加深色蒙层

- 封面图：绝对定位在左侧，有圆角和阴影

- 右侧信息：番剧名（中日双语）、评分（大字显示）、类型标签、状态（完结/连载中）

- 收藏按钮：已收藏显示\"取消收藏\"，未收藏显示\"加入收藏\"，调用
  favStore

详情内容区

- 简介：多行文字，超过 4 行折叠显示，点击\"展开\"查看全文

- 信息表格：首播日期、完结日期、集数、每集时长、来源（原作/漫改/游戏改等）、制作公司

- 类型标签列表：动作、冒险等标签，点击标签可跳转搜索页搜索该标签

- 角色列表：展示主要角色的立绘和名字（AniList 提供 characters 字段）

模块四：收藏页（/favorites）

- 展示所有已收藏的番剧列表（从 favStore 读取）

- 卡片样式同首页，但右上角有删除收藏的按钮

- 空状态：无收藏时显示提示 + 跳转首页的按钮

- 收藏数据存 localStorage，刷新页面不丢失（pinia-plugin-persistedstate
  自动处理）

模块五：导航栏

- 左侧：Logo + 项目名称 AniVault

- 右侧：首页、搜索、收藏 三个导航链接，收藏旁边显示数量 badge

- 当前页面对应的导航项高亮（用 router-link-active class）

- 移动端：导航折叠为汉堡菜单（做响应式是加分项）

AniList GraphQL 请求示例

+----------------------------------------------------------------+
| **代码参考**                                                   |
|                                                                |
| // 封装在 utils/anilist.js 中                                  |
|                                                                |
| const BASE_URL = \"https://graphql.anilist.co\"                |
|                                                                |
| // 获取当季新番                                                |
|                                                                |
| export async function getCurrentSeason() {                     |
|                                                                |
| const query = \`                                               |
|                                                                |
| query (\$season: MediaSeason, \$year: Int) {                   |
|                                                                |
| Page(page: 1, perPage: 20) {                                   |
|                                                                |
| media(season: \$season, seasonYear: \$year, type: ANIME, sort: |
| POPULARITY_DESC) {                                             |
|                                                                |
| id title { romaji native } coverImage { large }                |
|                                                                |
| averageScore episodes status                                   |
|                                                                |
| }                                                              |
|                                                                |
| }                                                              |
|                                                                |
| }                                                              |
|                                                                |
| \`                                                             |
|                                                                |
| const res = await axios.post(BASE_URL, {                       |
|                                                                |
| query,                                                         |
|                                                                |
| variables: { season: \"WINTER\", year: 2025 }                  |
|                                                                |
| })                                                             |
|                                                                |
| return res.data.data.Page.media                                |
|                                                                |
| }                                                              |

| **当季度计算**                                                 |
|                                                                |
| 当前季度判断方法：                                             |
|                                                                |
| 1月-3月 → WINTER，4月-6月 → SPRING                             |
|                                                                |
| 7月-9月 → SUMMER，10月-12月 → FALL                             |
|                                                                |
| 可以用 new Date().getMonth() + 1                               |
| 判断当前月份，然后映射到对应季度。                             |
|                                                                |
| 年份直接用 new Date().getFullYear()。                          |

常见问题 & 注意事项

- AniList 频率限制：每分钟 90
  次请求，正常开发不会触碰，但如果疯狂刷新调试要注意

- 图片懒加载：番剧封面图较多，给 img 标签加 loading=\"lazy\"
  属性，避免首屏请求太多图片

- 中文名称：AniList 的 title.chinese 字段不总是有数据，优先取
  title.chinese，没有则 title.romaji

- 详情页刷新问题：用户直接访问 /anime/123 时需要用 route.params.id
  重新请求数据，不能依赖跳转时传入的数据

- GraphQL 报错处理：AniList 就算报错也返回 200 状态码，错误信息在
  res.data.errors 中，需要额外判断

- Vercel 部署刷新 404：Vue Router History 模式在 Vercel 上需要配置
  vercel.json 的 rewrites，把所有路由指向 index.html
